AI Sales Copilot - System Design & Architecture Document

*** CURRENT PROTOTYPE IMPLEMENTATION STATE ***
Note: To validate the product quickly and achieve low latencies, we are running an optimized consolidated architecture.
Instead of a distributed set of microservices communicating over NATS, the pipeline is integrated into:
- Web Audio Mixer: Merges Rep Mic (Channel 0) and Customer Speaker Loopback (Channel 1) in the browser.
- STT Proxy (services/stt-proxy/server.js): Connects to Deepgram Live Stream (diarize: false, multichannel: true, channels: 2).
- Transcript Aggregator (services/transcript-aggregator/Aggregator.js): Reconstructs sentences using Punctuation, VAD/Speech Final, and watchdog timers.
- Event Detector (services/event-detector/detector.js): Runs local gatekeepers (filler, greeting, question-answering context, local Zero-Shot classification) and Groq Llama 3.1 8B (LLM 1) for intent classification.
- Suggestion Generator (services/stt-proxy/server.js): Integrates local playbook facts, executing real-time Tavily competitor searches and Groq Llama 3.1 8B (LLM 2) to generate direct response cues.
- React HUD (packages/ui/src/App.jsx): Performs Jaccard similarity deduplication, plays synthesized Web Audio API chimes, and runs a browser-local Transformers.js embedding model (all-MiniLM-L6-v2) to handle semantic check-offs.

Component Details

1. Streaming STT Service
   - Status: Implemented
   - Enforces `diarize: false`, `multichannel: true`, and `channels: 2` to isolate Rep (mic) on Channel 0 and Customer (speaker) on Channel 1.
   - Prevents AI clustering speaker role inversion bugs.

2. Transcript Aggregator (services/transcript-aggregator/Aggregator.js)
   - Status: Implemented
   - Buffers speaker-attributed text strings.
   - Finalizes utterances based on:
     1. Punctuation Rule: Sentence ending in `.`, `!`, `?` (Immediate).
     2. Speech Final Rule: Deepgram speech activity silence event detected (500ms).
     3. Watchdog Timer: Fallback to flush buffer if no activity occurs for 3000ms.

3. Event Detector (services/event-detector/detector.js)
   - Status: Implemented
   - Employs a low-latency gatekeeper funnel before hitting cloud LLM services:
     1. Question-Answering context: Traverses history to see if the Rep's last utterance was a question. If true, overrides the filler word filter so answers like "yeah" are not discarded.
     2. Filler Check: Short text blocks (<4 words) containing only fillers (yeah, ok, etc.) are bypassed (cards persist).
     3. Neutral / Greeting Check: Bypasses hello greetings and school identifier confirmations.
     4. Keyword bypass: If sales/objection/competitor keywords are in the text, instantly routes to LLM 1.
     5. Local Zero-Shot classification: Uses Xenova/nli-deberta-v3-small to measure small-talk confidence. If >= 70%, exits early.
   - Routinely sends bypassed texts to Groq Llama 3.1 8B to classify sales intents using neutralized, shortened categories (FEES, DEGREE, PLACEMENT, etc.).

4. Memory Lane (Session History Cache)
   - Status: Implemented
   - Utilizes a sliding memory array of the last 10 utterances.
   - Re-applies active Rep/Customer speaker roles dynamically based on the current session's `repSpeakerId` flag.

5. Reflex Lane (Playbook Guidelines)
   - Status: Implemented
   - Extracts facts from `PLAYBOOK_FACTS` mapped directly to active intents. Filters out guideline text that is irrelevant to the active classification, reducing LLM 2 prompt token overhead.

6. Hybrid Lane (Real-Time Web Search)
   - Status: Implemented
   - Triggered when competitor is classified but the entity name is not listed in our static playbook facts.
   - Queries the Tavily Search API and returns the top 2 content snippets, which are injected into the prompt.

7. Suggestion Generator (LLM 2)
   - Status: Implemented
   - Queries Groq Llama 3.1 8B using playbook guidelines and search snippets.
   - Rules: Outputs 2-3 bullets maximum, formatting responses strictly as direction cues with a single suggested respond phrase in parentheses: `(Say: "...")`.
   - Incorporates real-time facts directly into the suggested comparison phrasing.

8. Suggestion Deduplicator (Jaccard Overlap)
   - Status: Implemented in packages/ui/src/App.jsx
   - Compares incoming cards against active cards on screen using a Jaccard word-similarity index.
   - Ignores filler words and maps currency variations (e.g. lakh, rs). If similarity is > 55%, the card is discarded to prevent alert fatigue.

9. Suggestion Check-off (Semantic Completion)
   - Status: Implemented in packages/ui/src/App.jsx
   - Generates local embeddings of Rep's spoken sentences using Xenova/all-MiniLM-L6-v2.
   - Computes cosine similarity against active suggestions. If > 0.65, marks the card as completed and slides it off screen.

Security & Tenant Isolation Plans (Deferred for Production)
- Zero Retention: Audio streams are processed in-memory and are never written to disk.
- Tenant keys: Redis/Postgres keys will incorporate a prefix format of `tenant_id:call_id`.
- NATS Subjects: Subjects will follow a `tenant.<id>.call.<id>` subject hierarchy.
