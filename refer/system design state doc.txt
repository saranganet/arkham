AI Sales Copilot - System Design & Architecture Document

*** CURRENT PROTOTYPE DEVIATION ***
Note: To validate the product quickly, we are currently running a simplified prototype architecture. 
Instead of the full distributed Event Detector -> NATS -> Lanes pipeline, the current integrated flow is:
Local Mic + Shared Tab Audio -> Browser Web Audio Mixer (Stereo Merger) -> STT Proxy -> Transcript Aggregator -> Event Detector (detector.js) -> LLM Suggestion (Groq/Gemini) -> WebSocket Gateway -> React UI (Timeline Layout)

Key Prototype Milestones Completed:
  - Premium Cluely-style React 19 Frontend with canvas level meter visualizer and interactive checklist.
  - Multi-channel stereo mixing (mic in Left channel, tab loopback audio in Right channel) providing 100% accurate speaker separation.
  - Dynamic playbook selection parameterizing the STT proxy with distinct prompt guidelines, specifically customized with the "Newton School Bangalore" admissions manual.
  - Hybrid Event Detector (services/event-detector) implementing 0ms early exit gatekeepers (filler filters, neutral greeting filter) and regex rules alongside a local Llama 3.1 8B parameter model (Ollama) or cloud fallback.
  - Single tactical suggested response phrasings in direct (Say: "...") format, replacing legacy "Soft" vs "Bold" variations.
  - Clean vertical timeline queue (no card boxes or background panels) for displaying incoming cues one-by-one.
  - Jaccard similarity word overlap deduplicator in the React UI to prevent duplicate suggestions and alert fatigue.
  - Real-Time Visual/Audio Cue Alerts synthesizing a clean 880Hz electronic sound chime via Web Audio API, and a custom CSS keyframe glow animation.

High-Level Architecture
Exotel ↓ Audio Stream ↓ Streaming STT ↓ Transcript Aggregator ↓ Event Detector ↓ NATS Event Bus ↓ AI Lanes ↓ Suggestion Ranker ↓ WebSocket Gateway ↓ Chrome Extension

Core Architecture Principles
Human-First Assistance
The AI assists the rep.
The AI does not replace the rep.

Direction Over Scripts
The system should generate:
Direction + Supporting Context
The system should not generate:
	•	Long responses
	•	Read-aloud scripts
	•	Complete paragraphs
Example:
Talk about implementation timeline
Salesforce → consulting-heavy
HubSpot → faster onboarding
Exact rendering intentionally undecided.

Event-Driven Architecture
The system reacts to events.
Not transcript chunks.
Not every sentence.
Not every utterance.
Architecture:
Transcript ↓ Event Detector ↓ Event ↓ Lane ↓ Suggestion

Backend-Centric Intelligence
Chrome Extension = UI only.
All intelligence remains on backend.
