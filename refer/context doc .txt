AI Sales Copilot - Project Context Document
Current Status
* PROTOTYPE COMPLETE: We have built a fully functional prototype. The frontend is a premium React/CSS dashboard supporting hardware-level speaker distinguishing (mixing local mic and tab loopback audio into a stereo stream for 100% accurate diarization) and single-mic AI diarization. When the customer speaks, the backend aggregates transcriptions, passes them to a Cognitive-First local Event Detector (implementing 0ms/20ms local Zero-Shot nli-deberta-v3-small classification to bypass LLM calls for general small talk), and routes active objections to the Groq dual-LLM pipeline (Llama 3.1 8B Tier 1, Llama 3.3 70B Tier 2) to formulate concise point-based tactical response cues in direct format. Cues are rendered in a clean, vertical timeline UI with auto-scroll and cue persistence during customer backchanneling, utilizing a normalized Jaccard word similarity deduplication algorithm to prevent duplicates and alert fatigue.

Product Vision
Build a real-time AI Sales Copilot that assists sales representatives during live sales calls.
This is NOT a transcription product.
This is NOT a meeting note-taking product.
The purpose of the system is to help sales reps perform better during live conversations.
Examples:
	•	Competitor battlecards
	•	Objection handling
	•	Product knowledge assistance
	•	Discovery guidance
	•	Deal coaching
	•	Call memory
	•	Real-time sales intelligence
Comparable vision:
"Cluely for Sales"

Core Moat
The core moat is not transcription.
The core moat is not LLM access.
The core moat is becoming the intelligence layer that removes everything except the conversation itself.
The ideal future state:
The sales rep only needs to focus on the customer conversation.
The system handles:
	•	Remembering details
	•	Tracking discovery
	•	Tracking objections
	•	Tracking competitors
	•	Tracking buying signals
	•	Surfacing battlecards
	•	Surfacing product knowledge
	•	Surfacing relevant context
	•	Surfacing next conversation directions
The AI acts as a second brain for the salesperson.
The salesperson focuses only on human interaction.

Core Product Principles
	•	Human-first assistance
The AI assists the rep.
The AI does not replace the rep.

	•	Preserve authenticity
The system should avoid creating scripted conversations.

	•	Direction over scripting
The system should guide thinking.
The system should not generate responses to be read verbatim.

	•	Intelligence lives on backend
The extension is only a UI layer.

	•	Attention is scarce
Most transcript should generate no suggestion.
Only important moments deserve rep attention.

AI Lane Architecture
Lane 1 - Reflex Lane
Purpose:
Instant reactions.
Examples:
	•	Competitor detection
	•	Objection detection
	•	Battlecard retrieval
No LLM.
Target latency:
<100ms

Lane 2 - Hybrid Lane
Purpose:
Handle unknown competitors and unknown entities.
Uses LLM.
Generates temporary battlecards.
Caches results.

Lane 3 - Tactical Lane
Purpose:
Answer product questions.
Uses:
	•	Qdrant
	•	RAG
	•	Product documentation
	•	Security documentation
	•	Pricing documentation

Lane 4 - Strategic Lane
Purpose:
Coach the rep.
Examples:
	•	Budget not discussed
	•	Decision maker missing
	•	Timeline missing
	•	Buying intent increasing
Runs periodically.

Lane 5 - Memory Lane
Purpose:
Maintain call state.
Tracks:
	•	Budget
	•	Timeline
	•	Competitors
	•	Risks
	•	Objections
	•	Decision makers
	•	Buying signals
All other lanes read from memory.

Event Architecture Philosophy
The system should not react to transcript chunks.
The system should react to meaningful events.
Wrong:
Transcript ↓ Lane ↓ Suggestion
Correct:
Transcript ↓ Event Detector ↓ Event ↓ Lane ↓ Suggestion
