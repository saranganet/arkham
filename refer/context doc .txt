AI Sales Copilot - Project Context Document

Current Status
* PROTOTYPE COMPLETE & OPTIMIZED: We have built a fully functional prototype. The frontend is a premium React/CSS dashboard supporting hardware-level speaker separation (enforcing 2-channel stereo mapping where channel 0 is the microphone/Rep and channel 1 is the speaker loopback/Customer, completely disabling AI diarization). When either the Customer or Rep speaks, the backend aggregates transcriptions, runs local 0ms/20ms early-exit gatekeepers (filler filters, greeting filters, and local Zero-Shot nli-deberta-v3-small classification to bypass LLM calls for pleasantries), and runs a dual-LLM pipeline utilizing Groq Llama 3.1 8B (llama-3.1-8b-instant) for both intent classification (LLM 1) and suggestion generation (LLM 2) to reduce cloud costs by 12x and latency to ~100ms. The system integrates a Qdrant-powered RAG pipeline (with local mock fallback) and manager direct rule overrides (direct_rules.json) to retrieve context. If no RAG results are found or an unknown competitor is mentioned (e.g., Coding Ninjas), the backend queries the Tavily API in real-time, injecting the web results into the prompt. Category keys have been shortened and neutralized to topic-focused codes (FEES, PLACEMENT, DEGREE, BUY_SIGNAL, etc.) to handle both objections and inquiries neutrally, and are mapped to friendly names (CATEGORY_MAP) to prevent code leakage. Cues are rendered in a clean, vertical timeline UI with auto-scroll and cue persistence, utilizing a local semantic check-off via Transformers.js (cosine similarity > 0.65) and a normalized Jaccard word similarity deduplication algorithm.

AI Lane Architecture (Prototype Implementation)
In the high-performance prototype, the distributed lanes have been consolidated into a streamlined, high-speed dual-LLM pipeline running on the stt-proxy server to achieve a sub-350ms end-to-end response time.

1. Reflex Lane (Consolidated):
   - Handles objection and competitor matches by checking manager direct rule overrides or querying Qdrant RAG (with local mock fallback) using local MiniLM embeddings. Bypasses LLM calls for small talk using local regex/zero-shot checks.
   - Target latency: <20ms.

2. Hybrid Lane (Consolidated):
   - Triggers automatically when a competitor intent (`COMPETITOR`) is classified by LLM 1 but the competitor entity name is not present in our local database.
   - Executes real-time Tavily search queries to get pricing, curriculum, and reviews, then pipes the snippets directly into LLM 2 for comparison card generation.

3. Tactical Lane (Consolidated):
   - Mapped directly to the Suggestion Generator (LLM 2) prompt context.
   - Provides concrete direction cues and single direct suggested responding phrases in parentheses `(Say: "...")`.

4. Strategic Lane (Consolidated):
   - Mapped to the proactive topic detector rules. If the Rep proactively mentions a topic (e.g. "let's discuss pricing"), the system bypasses filler checks and generates relevant prompt cues to guide the discussion.

5. Memory Lane (Consolidated):
   - Managed via the sliding `conversationHistory` window containing speaker-attributed utterances, feeding the context window of LLMs 1 and 2 to resolve pronoun references and maintain conversation state.
